# Week 1 Assignment: Markdown Introduction

## Instructions

Create a file called `introduction.md` in this repository with the following sections:

### Required Content

1. **Your Name** (Heading 1)
2. **About Me** (Heading 2)
   - Write a paragraph about yourself
3. **Research Interests** (Heading 2)
   - Use a bulleted list with 3-5 interests
4. **Why I'm Taking This Course** (Heading 2)
   - Write a paragraph explaining your goals
5. **Related Resource** (Heading 2)
   - Add a link to a website, article, or DH project relevant to your interests
6. **Bash Tutorial Practice** (Heading 2)
   - Paste a transcript of 5-10 commands you ran while working through the [Bash tutorial](https://programminghistorian.org/en/lessons/intro-to-bash)
   - Use a code block (three backticks) to format the transcript
   - **Tip**: After completing the tutorial, run `history` in your terminal to see all your commands, then copy the interesting ones
   - Include a variety of commands (e.g., `pwd`, `ls`, `mkdir`, `cd`, `cat`, etc.)
   - You can include just the commands, or commands with output - either is fine

### Requirements

- Use proper Markdown syntax
- Include at least one **bold** word
- Include at least one *italic* word
- Include a code block with your CLI transcript
- Push your completed `introduction.md` file to GitHub before the deadline

### Grading

- Completeness: 50% (all required sections present, including CLI transcript)
- Markdown syntax: 30% (proper use of headings, lists, links, bold/italic, code blocks)
- GitHub workflow: 20% (properly committed and pushed)
